🔤 专业术语解释

🕐 游戏刻（Game Tick）

定义： Minecraft 服务器的基本时间单位，每秒运行 20 个游戏刻。

换算： 1 游戏刻 = 0.05 秒，20 游戏刻 = 1 秒

示例： 如果某个延迟设置为 100，表示延迟 5 秒（100÷20=5）

📦 材质名称（Material）

定义： Minecraft 中每个物品和方块的唯一标识符。

示例： DIAMOND_SWORD（钻石剑）、IRON_INGOT（铁锭）

注意： 不同版本的材质名称可能不同，插件会自动适配。

🎯 坐标系统

X 轴： 东西方向（正数向东，负数向西）

Y 轴： 上下方向（正数向上，负数向下）

Z 轴： 南北方向（正数向南，负数向北）

⚠️ 重要提醒： 所有时间相关的配置（如资源刷新速度、火焰弹延迟等）都必须使用整数，因为它们以游戏刻为单位。浮点数会被强制转换为整数！

3. ⚡ GeneratorsConfig.yml（生成器配置）

💎 资源生成器详解

生成器是起床战争的核心机制，负责生产铁锭、金锭、钻石和绿宝石等资源。

🔩 铁锭生成器

Default: iron-delay: 2 # 生成延迟：2游戏刻 = 0.1秒 iron-amount: 2 # 每次生成数量：2个铁锭 iron-spawn-limit: 32 # 最大堆叠数量：32个

计算： 每 0.1 秒生成 2 个铁锭，最多堆叠 32 个。

🏆 金锭生成器

gold-delay: 6 # 生成延迟：6游戏刻 = 0.3秒 gold-amount: 2 # 每次生成数量：2个金锭 gold-spawn-limit: 7 # 最大堆叠数量：7个

💎 钻石生成器（三级升级）

# 一级钻石生成器 diamond-tier-I-delay: 30 # 30游戏刻 = 1.5秒 diamond-tier-I-amount: 1 # 每次1个钻石 diamond-tier-I-spawn-limit: 4 # 二级钻石生成器（游戏开始6分钟后激活） diamond-tier-II-delay: 20 # 20游戏刻 = 1秒 diamond-tier-II-start: 360 # 360游戏刻 = 18秒后开始 # 三级钻石生成器（游戏开始18分钟后激活） diamond-tier-III-delay: 15 # 15游戏刻 = 0.75秒 diamond-tier-III-start: 1080

说明： 钻石生成器会随着游戏进程自动升级，生成速度越来越快。

4. ✨ SpecialItemsConfig.yml（特色功能配置）

🎉 BedWars1058-XP 独有功能： 以下四个特色功能是本增强版独有的，为游戏增添了更多策略性和趣味性！

📜 回城卷轴（WarpPowder）

warp-powder: teleport-time: 3 # 传送延迟：3秒 delay: 15 # 使用冷却：15秒

功能： 允许玩家快速传送回基地，传送过程中会显示粒子效果。

注意： 传送过程中受到伤害会被打断，物品会返还给玩家。

🛡️ 自救平台（RescuePlatform）

rescue-platform: delay: 15 # 使用冷却：15秒 break-time: 150 # 平台持续时间：150游戏刻 = 7.5秒 material: SLIME_BLOCK # 平台材质：粘液块

功能： 在玩家脚下生成临时平台，防止掉入虚空。

图案： 生成十字形平台，自动检测是否可放置。

🏰 防守墙（ProtectionWall）

protection-wall: delay: 150 # 使用冷却：150游戏刻 = 7.5秒 width: 5 # 墙体宽度：5格 height: 3 # 墙体高度：3格 distance: 2 # 距离玩家：2格 material: CUT_SANDSTONE # 墙体材质：切制砂岩

功能： 在玩家面前生成防御墙体，可阻挡敌人攻击。

智能： 根据玩家朝向自动调整墙体方向。

⭐ 经验起床系统（XP System）

xp-config: iron-value: 1 # 铁锭经验值：1点 gold-value: 2 # 金锭经验值：2点 diamond-value: 5 # 钻石经验值：5点 emerald-value: 10 # 绿宝石经验值：10点 exp-bottle-value: 0 # 经验瓶经验值：0点（不转换）

功能： 拾取资源时获得经验值而非物品，为游戏增加新的策略维度。

启用： 在竞技场配置中设置 enableXP: true 来启用此功能。

⚠️ 重要规则： 如果某个资源的经验值设置为 0，该资源将不会被转换为经验，玩家拾取时仍然获得原始物品。

转换示例：

# 示例配置 xp-config: iron-value: 2 # 铁锭转换为2点经验 gold-value: 5 # 金锭转换为5点经验 diamond-value: 0 # 钻石不转换，保持原物品 emerald-value: 15 # 绿宝石转换为15点经验

在上述配置中，玩家拾取钻石时会直接获得钻石物品而不是经验值，其他资源则按设定比例转换为经验。

⭐ 经验系统详解

🎓 经验起床系统介绍

经验起床系统是 BedWars1058-XP 的核心创新功能，将传统的资源收集转换为经验积累机制，为游戏带来全新的策略深度。

🔧 系统工作原理

1. 资源转换机制

当玩家拾取资源时，系统会根据配置将资源转换为经验值：

  • 转换条件： 竞技场启用经验系统 + 资源经验值 > 0
  • 转换公式： 获得经验 = 资源数量 × 单个资源经验值
  • 视觉效果： 播放经验收集音效，显示经验增加

2. 转换规则详解

经验值设置 转换结果 玩家获得 适用场景
大于 0（如：iron-value: 2） ✅ 转换为经验 经验值 基础资源
等于 0（如：diamond-value: 0） ❌ 不转换 原始物品 珍贵资源
未配置 ❌ 不转换 原始物品 特殊物品

⚙️ 配置示例与策略

策略一：全资源转换

# 适合纯经验模式服务器 xp-config: iron-value: 1 # 铁锭 → 1经验 gold-value: 3 # 金锭 → 3经验 diamond-value: 10 # 钻石 → 10经验 emerald-value: 25 # 绿宝石 → 25经验

特点： 所有资源都转换为经验，玩家通过经验等级解锁物品。

策略二：混合模式

# 平衡经验与物品的混合模式 xp-config: iron-value: 2 # 铁锭转换为经验 gold-value: 4 # 金锭转换为经验 diamond-value: 0 # 钻石保持物品 emerald-value: 0 # 绿宝石保持物品

特点： 基础资源转经验，高级资源保持物品形态，增加策略选择。

策略三：传统模式

# 保持传统游戏模式 xp-config: iron-value: 0 # 所有资源都不转换 gold-value: 0 diamond-value: 0 emerald-value: 0

特点： 关闭经验转换，保持原版游戏体验。

🔧 常见问题解决

❓ 配置相关问题

Q: 为什么我设置的延迟时间不准确？

A: 请确保使用整数值。所有时间相关配置都以游戏刻为单位（20游戏刻=1秒）。浮点数会被强制转换为整数。

# 正确写法 delay: 100 # 5秒延迟 # 错误写法 delay: 5.5 # 会被转换为 5，实际延迟 0.25 秒

Q: 特殊物品不显示在商店中？

A: 检查以下设置：

  • 确认在 ShopConfig.yml 中启用了对应物品
  • 检查物品的材质名称是否正确
  • 确保商店分类配置正确

Q: 经验起床功能不工作？

A: 请确保：

  • 在竞技场配置中设置 enableXP: true
  • 检查 XpConfig.yml 中的经验值设置
  • 重启服务器或重载配置

Q: 经验起床功能中，为什么有些资源没有转换为经验？

A: 检查 XpConfig.yml 中对应资源的经验值设置：

  • 经验值 > 0： 资源会按比例转换为经验值
  • 经验值 = 0： 资源不会转换，玩家获得原始物品
  • 未配置： 默认不转换，获得原始物品
# 示例：部分转换配置 xp-config: iron-value: 2 # 铁锭转换为经验 gold-value: 5 # 金锭转换为经验 diamond-value: 0 # 钻石保持原物品 emerald-value: 0 # 绿宝石保持原物品

这种配置适合希望保留高级资源作为物品，而将基础资源转换为经验的服务器。

🎮 游戏运行问题

Q: 地图设置完成后无法启用？

A: 检查以下必要设置：

  • 至少创建了2个队伍
  • 每个队伍都设置了出生点和床位置
  • 设置了等待大厅出生点
  • 添加了必要的资源生成器

Q: 玩家无法加入游戏？

A: 可能的原因：

  • 地图未启用：使用 /bw enableArena [地图名]
  • 权限不足：检查玩家是否有加入权限
  • 服务器模式配置错误：检查 serverType 设置
